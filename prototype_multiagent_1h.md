# Прототип за ~1 час: мини-оркестрация из двух «агентов» (Python)

Цель: получить **запускаемый репозиторий** и честную историю для отклика: «собрал минимальную мультиагентную схему: два шага с разными ролями/промптами и общим контекстом». Это не прод и не «рой из десятков агентов», но закрывает пробел **лучше, чем ноль**, если честно назвать уровень: *proof of concept за вечер*.

Опора на ваши навыки: Python из резюме, Docker/Linux, привычка доводить до «работает».

**Готовый отдельный проект в этом workspace:** `local-multiagent-chat/` - FastAPI + Ollama + одна HTML-страница, два шага (планировщик → исполнитель). Его можно копировать в отдельный репозиторий на GitHub или подменять UI/оркестратор кодом из **ai-chat** / **test-tracker-mvp** (см. README внутри проекта).

---

## Что считать «готово» за час

1. Один скрипт или модуль, который **последовательно** вызывает две логические роли:
   - **Agent A** - уточняет или декомпозирует задачу (один системный промпт).
   - **Agent B** - по выводу A даёт итоговый ответ или план (второй промпт).
2. Общий контекст передаётся **явной строкой** (без сложной памяти).
3. Запуск: `python main.py "ваш вопрос"` или через `.env` с ключом API.
4. В `README.md` - 5-7 строк: что это, как запустить, чем отличается от одного вызова LLM.

Опционально (+15-20 минут): третий шаг **critic** - короткая проверка ответа B.

## Логика `main.py` (идея, не полный код)

1. Загрузить `OPENAI_API_KEY` из окружения.
2. **Шаг 1 (Agent A - planner):** системный промпт вроде: «Ты планировщик. Разбей запрос пользователя на 2-3 шага, без лишней воды».
3. Передать ответ A в **шаг 2 (Agent B - executor):** «Ты исполнитель. По плану ниже дай финальный ответ пользователю».
4. Вывести в консоль: сначала план (опционально), затем финальный ответ.

Так вы демонстрируете **оркестрацию** (порядок, разные роли, передача состояния), без обмана про AutoGen/Crew «рой».
